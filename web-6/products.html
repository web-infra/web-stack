OfficeConnector Server

The OfficeConnector Server is setup in the customer's office, and consists of the following modules:

Email:

Email server with support for IMAP, POP3 & SMTP protocols. It provides automatic Email Archival of all incoming & outgoing emails, or selectively, based on user. It can fetch (pull) email from a hosted (cloud) email service like Gmail or Yahoo, or act as a MX host, with spam/virus filtering. It can also accept email relayed by a service like OfficeConnector Clean Email Relay. It has a Groupware module, which provides shared calendars and address book. Users can create events, invite others and check their free-busy timings. Mozilla Thunderbird can be used as a feature rich email client on Windows and Linux desktops. Used in this fashion, Mozilla Thunderbird with OfficeConnector Server (Groupware) can be considered in place of (or as a replacement for) Outlook+Exchange, but at a far lower cost.

Mobile Sync: With the Groupware module, users can synchronize their calendars and address books with their mobile phones (Android, Blackberry), periodically and automatically. Email can be accessed using IMAP, and users can view their email and online folders too.

Outlook Sync: Users can synchronize their calendars and address books with their desktop Outlook client.

Firewall:

Provides network level packet filtering with stateful inspection, and facilities like PAT/NAT, etc. Control Internet (browsing) access based on sophisticated controls based on time, internal system, destination domains, username, or page content (i.e. Content filtering). Block (yahoo, gtalk) chat, private email (yahoo, gmail) sites, social networking sites like Facebook. Browsing traffic is scanned for viruses. Reports on browsing & network traffic available. In case of multiple links, traffic distribution and link failover are possible.

VPN:

SSL based VPN which can be used to connect users to the head office or multiple offices (site-to-site), securely. Users will be provided with certificates to connect, and can additionally be authenticated with username/password.

File Server:

Acts as a central file repository for a network of Windows (TM) and Linux desktops, with user authentication and access permissions. It can also act as a Domain Controller, and provide central authentication, thereby allowing users to login from any system on the network, and get access to network resources with the set permissions. Data backups of desktop shares can also be taken on the Server's hard disk. Off site data backups are also possible.

It is therefore a complete Intranet server (like an SMB server) for SMBs and can be considered as an alternative to (or to replace) Microsoft (R) Small Business Server (TM).

OfficeConnector High Availability Cluster

The HA Cluster is a cost-effective solution for environments where a very high level of uptime is required. The simple idea is that 2 low cost systems can together provide better uptime than a single very high cost, high end server; or 2 servers can be clustered together to ensure that a failure in either doesn't lead to service downtime.

  • Two systems are required to setup the cluster; they must have at least 2 NICs.
  • One NIC on each connects to the network; the other is conneted to the other server over a dedicated (crossover) connection. The servers are setup with identical services, etc
  • The servers can store data on their own HDDs and have them replicated if feasible; or access data from NAS/SAN.
  • The Secondary server continuously monitors the Prinary over the dedicated connection; if it detects any failure of the Primary or of any service, it takes over the complete function of the Primary or of the failed service. In case of full takeover, it activates itself on the LAN with the Primary's IP address.
  • Once the Primary is up, the cluster reverts to its original state.

OfficeConnector Request Tracking System

Many critical activities are happening in your company. A system dependent work environment can be created by using RTS, which also aids in tracking the processes. Management can get updated without having to call people. It is ideal for departments like customer support, admin, HR, facilities/plant maintenance, etc.

  • The application is accessible with a standard web browser.
  • Requests can come by web interface, email or SMS and go into the appropriate queue.
  • Configurable auto replies with unique ticket/docket number in the Subject line. All further communications with this number automatically go to the correct ticket.
  • Multiple roles  Staff working on the ticket, Managers with access privileges.
  • Tickets can be assigned to staff; all email exchanges are recorded; Comments, not visible to the requester, can be added.
  • Tickets can be transferred to others if required; the complete history also moves with it.
  • Time based escalations are possible.
  • Various reports on staff performance, etc. can be generated.
